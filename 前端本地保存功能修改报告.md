# 前端本地保存功能修改报告

## 1. 根本原因

之前保存时提示"缺少后端接口"，是因为：
1. 前端调用了 `api_client.save_screenplay_json()` 方法
2. 但该方法对应的后端 `PUT /artifacts/screenplay_json` 接口已被回退删除
3. 导致保存时抛出 `ApiClientError`，提示缺少后端接口

本次改为前端本地保存，不依赖后端接口。

## 3. 本地保存机制

### 存储 key

```python
_key = f"screenplay_edits_{project_id}"
```

按 `project_id` 区分，避免不同项目数据混淆。

### 保存位置

- **session_state**：`st.session_state[f"screenplay_edits_{project_id}"]`
- **本地 JSON**：通过 `storage.update_project()` 持久化到 `data/projects.json`

### 加载优先级

```
1. 请求后端 screenplay_json
2. 检查 localStorage（session_state / projects.json）
3. 如果有本地编辑版本，优先展示本地版本
4. 如果没有，展示后端原始版本
```

### 恢复原始版本

每个页面顶部显示：
- 如果有本地编辑：`st.info("当前显示的是本地编辑版本")` + "恢复后端原始版本"按钮
- 点击恢复：清除本地数据，重新加载后端原始版本

## 5. ID 自动生成规则

| 类型 | 格式 | 规则 |
|---|---|---|
| 人物 | `char_001` | 最大编号 + 1 |
| 场景 | `scene_001` | 最大编号 + 1 |
| 情节 | `event_001` | 最大编号 + 1 |

## 6. 重名校验

- 去首尾空格
- 不能为空
- 不能与其他同类数据重复（排除自身）
- 大小写不敏感

## 7. 关系名称映射

- 人物关系：`char_id` → `character.name`（实时从当前数据构建映射）
- 场景关系：`scene_id` → `scene.title`（实时映射）
- 因果关系：`event_id` → `event.title`（实时映射）

编辑名称后，关系展示自动同步更新。

## 9. 注意事项

- 当前保存只是前端本地保存，不会写入 `backend/data` 文件
- 本地数据存储在浏览器 session_state 和本地 `data/projects.json` 中
- 如果清除浏览器缓存或删除 `data/projects.json`，本地编辑会丢失
- 未来如果需要多人协作或真正写入项目文件，再补后端保存接口
