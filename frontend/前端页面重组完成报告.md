## 4. 状态管理方式

### projectId 共享

- 所有页面通过 `st.session_state["current_project_id"]` 获取当前项目 ID
- 进入编辑器时从本地 JSON 持久化恢复到 session_state（editor.py 第 30-43 行）
- 页面刷新后状态不丢失

### screenplay_json 数据加载

采用 **方案 B：统一缓存**，通过 `frontend/utils/data_loader.py` 实现：

```python
# 共享加载函数
load_screenplay_data(project)    # 加载 screenplay_json
load_screenplay_yaml(project)    # 加载 screenplay_yaml
load_rendered_markdown(project)  # 加载渲染后 markdown
get_backend_project_id(project)  # 获取后端项目 ID
refresh_all_artifacts(project)   # 强制刷新全部制品
```

**三级缓存策略**：
1. 先查 project dict 缓存
2. 再查 session_state
3. 最后才发 HTTP 请求（仅当 backend_project_id 存在且缓存为空时）

拉取成功后同时更新 session_state 和本地持久化存储。
