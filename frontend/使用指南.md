# AI 小说转剧本工具 - 使用指南

## 项目简介

本项目是一个基于 Streamlit 的 Web 应用，用于将小说原文转换为剧本格式。支持项目管理、原文编辑、人物管理、场景管理和场次编排等功能。

---

## 目录结构

```
novel_to_script_app/
├── app.py                 # 应用入口
├── run.bat               # Windows 启动脚本
├── requirements.txt      # Python 依赖
├── data/
│   └── projects.json     # 项目数据存储
├── utils/
│   ├── storage.py        # 数据读写工具
│   └── state.py          # 状态管理工具
└── views/
    ├── home.py           # 项目首页
    ├── editor.py         # 编辑器布局
    ├── original.py       # 原文管理
    ├── characters.py     # 人物管理
    ├── scenes.py         # 场景管理
    └── acts.py           # 场次管理
```

---

## 启动方式

### 方式一：使用 run.bat（推荐）

双击运行 `run.bat` 文件，脚本会自动：
1. 检查 Python 环境
2. 启动 Streamlit 服务
3. 打开浏览器访问应用

### 方式二：命令行启动

```bash
# 进入项目目录
cd novel_to_script_app

# 启动应用
python -m streamlit run app.py
```

## 功能使用说明

### 一、项目首页

**功能**：管理所有剧本项目

**操作**：
- **新建项目**：点击右上角「➕ 新建项目」按钮，填写项目名称和简介
- **进入项目**：点击项目卡片的「进入项目」按钮
- **删除项目**：点击「🗑️ 删除」按钮，确认后删除（不可恢复）

---

### 二、原文管理

**功能**：编辑和保存小说原文

**路径**：左侧导航 → 📄 原文

**操作**：
1. **直接输入**：在文本框中输入或粘贴小说原文
2. **上传文件**：点击「上传 txt 原文文件」选择 txt 文件（支持 UTF-8/GBK 编码）
3. **字数限制**：最多 20000 字，超出会自动截断
4. **保存原文**：点击「💾 保存原文」按钮保存
5. **提交**：点击「🚀 提交」按钮（预留功能，暂未实现）

**注意**：
- txt 文件大小不能超过 100KB
- 页面底部显示当前字数统计

---

### 三、人物管理

**功能**：管理剧本中的角色

**路径**：左侧导航 → 👤 人物

**操作**：
1. **添加人物**：
   - 点击「+ 添加人物」卡片
   - 填写人物姓名、身份/角色、简介
   - 点击「✅ 添加」保存

2. **查看人物**：人物以竖向卡片形式展示，包含：
   - 头像占位（👤）
   - 人物姓名
   - 身份/角色
   - 简介

3. **操作按钮**：
   - 「🎨 生成形象」：预留功能
   - 「🖼️ 生成剧照」：预留功能
   - 「🗑️ 删除」：删除该人物

---

### 四、场景管理

**功能**：管理演出背景和地点环境

**路径**：左侧导航 → 🎭 场景

**概念说明**：
- **场景**：指演出背景或地点环境（如餐厅、皇宫、山洞等）
- 一个场景可以被多个场次重复使用

**操作**：
1. **添加场景**：
   - 点击「➕ 添加场景」展开表单
   - 填写场景名称、地点、场景描述
   - 点击「✅ 添加场景」保存

2. **查看场景**：场景以卡片形式展示，显示：
   - 场景名称
   - 地点
   - 场景描述
   - 被多少个场次使用

---

### 五、场次管理

**功能**：编排具体剧情单元

**路径**：左侧导航 → 🎬 场次

**概念说明**：
- **场次**：指具体剧情单元（如第 1 场：主角登场）
- 每个场次可以关联一个场景背景
- 每个场次可以添加多个出场人物

**操作**：

1. **添加场次**：
   - 点击「+ 添加场次」卡片
   - 自动创建默认场次（标题为「第 N 场」）
   - 进入编辑状态

2. **编辑场次**：
   - 点击场次卡片的「✏️ 编辑」按钮
   - 可编辑字段：
     - **场次标题**：场次名称
     - **关联场景**：选择该场次发生的场景（单选）
     - **出场人物**：选择该场次出场的人物（可多选）
     - **时间**：场次发生的时间（如夜晚、清晨）
     - **场次摘要**：剧情摘要
     - **场次内容**：剧本正文（预留）
   - 点击「💾 保存」保存并返回卡片列表
   - 点击「❌ 取消」放弃修改并返回

3. **查看场次**：场次卡片显示：
   - 场次标题
   - 关联场景名称
   - 出场人物列表
   - 时间
   - 摘要

## 数据存储

所有数据保存在 `data/projects.json` 文件中，包括：
- 项目基本信息（名称、简介、创建时间）
- 原文内容
- 人物列表
- 场景列表
- 场次列表

**注意**：
- 数据以 JSON 格式存储
- 无需数据库，直接读写文件
- 建议定期备份 `projects.json` 文件

---

## 注意事项

1. **浏览器兼容性**：推荐使用 Chrome、Edge、Firefox 等现代浏览器
2. **数据安全**：删除操作不可恢复，请谨慎操作
3. **字数限制**：原文最多 20000 字，超出部分会被截断
4. **文件编码**：上传 txt 文件建议使用 UTF-8 编码
5. **多用户**：当前版本为单机版，不支持多用户同时编辑

## 故障排除

### 问题：run.bat 无法启动

**解决**：
1. 检查 `D:\Tools\Miniconda3\envs\voicecal\python.exe` 是否存在
2. 修改 `run.bat` 中的 `PYTHON_EXE` 路径为实际 Python 路径

### 问题：页面加载缓慢

**解决**：
1. 检查网络连接
2. 尝试刷新页面
3. 重启应用

### 问题：数据丢失

**解决**：
1. 检查 `data/projects.json` 文件是否存在
2. 如有备份，恢复备份文件
3. 建议定期备份数据文件
