## Prompt Changes Required

### novel_reader.md — 新增 Chapter Reading Strategy

```
## Chapter Reading Strategy

1. **Prologue Detection:** If the first chapter's title contains "序"/"楔子"/"前传"
   and its text exceeds 2000 characters, treat it as `chapter_000`.
2. **Deep-Read Scope:** Deep-read chapters 0 (if exists) through 3.
   Extract all characters, events, relationships, foreshadowing, and continuity anchors.
3. **Skim Scope:** Skim chapters 4 and 5. Extract only major new characters
   and significant new events. Do NOT extract detailed events or minor characters
   from skim chapters.
4. **Screenplay Boundary:** The final screenplay will only cover deep-read chapters.
   Mark skim-chapter findings with `source_scope: "context_only"`.
```

### adaptation_planner.md — 新增 Screenplay Scope

```
## Screenplay Scope Constraint

- The screenplay must only adapt events from deep-read chapters
  (chapters marked with source_scope "deep_read" or unmarked chapters 0-3).
- Events from skim chapters (source_scope "context_only") provide background
  but must NOT appear in the scene_plan.
```

## E2E Tests

### Test 1: test_pipeline_e2e.py — run_v1() 直接调用

- 用预拆分的 6-8 章数据直接调用 `orchestrator.run_v1()`
- `save_intermediates=True`
- 断言:
  - job.status == "succeeded"
  - YAML 可解析、非空
  - screenplay_json 含 schema_version, scenes, story_bible, events
  - 无 "Fake Provider" / "当前项目主角" 占位符
  - audit_report 无 P0 错误
  - 中间产物全部落盘 (novel_analysis, adaptation_plan, screenplay_json)
  - chapter text 已 stripped（存 paragraph_count 而非全文）

### Test 2: test_full_flow_e2e.py — 完整链路

- 从 `docs/三生三世十里桃花.txt` 读取前 6-8 章原文
- ChapterIntakeService.auto_split_and_save() → chapters
- GenerationOrchestrator.run_v1() → YAML
- 同 Test 1 的结构断言
- 额外验证: 拆分出的章节数 ≥ 3
