# 人物/场景/情节持久化编辑修改报告

## 1. 根本原因

之前点击保存后显示没有更新或刷新后丢失，原因有三：

1. **后端没有保存接口**：`api_client.py` 中没有 PUT/PATCH 方法，前端编辑无法写入后端 data 文件
2. **前端只改本地 state**：编辑/添加/删除只调用 `storage.update_project()` 写入本地 `data/projects.json`，不调用后端 API
3. **页面数据优先读后端**：刷新后 `load_screenplay_data()` 从后端重新拉取未修改的 `screenplay_json`，覆盖前端本地修改

## 3. 新增/修改的后端接口

**接口路径**：`PUT /api/projects/{project_id}/artifacts/screenplay_json`

**请求体**：
```json
{
  "data": { "...完整 screenplay_json data..." }
}
```

**响应体**：
```json
{
  "id": "artifact_{project_id}_screenplay_json_v002",
  "type": "screenplay_json",
  "version": 2,
  "data": { "...保存后的 screenplay_json..." }
}
```

**写入文件位置**：`data/projects/{project_id}/artifacts/screenplay_json_v{NNN}.json`

**版本管理**：自动递增版本号（v001 → v002 → ...），旧版本保留不删除

## 5. ID 自动生成规则

| 类型 | 格式 | 规则 |
|---|---|---|
| 人物 | `char_001`, `char_002`, ... | 扫描现有 ID 中最大编号 + 1 |
| 场景 | `scene_001`, `scene_002`, ... | 同上 |
| 情节 | `event_001`, `event_002`, ... | 同上 |

如果已有 ID 不连续（如 char_001, char_005），新 ID 为 `char_006`。

## 6. 名称重复校验

- 人物：`name` 去首尾空格，不能为空，不能与其他人物重复（排除自身），大小写不敏感
- 场景：`title` 去首尾空格，不能为空，不能与其他场景重复（排除自身），大小写不敏感
- 情节：`title` 去首尾空格，不能为空，不能与其他情节重复（排除自身），大小写不敏感

重复时显示中文错误提示，不允许保存。

## 7. 持久化保存流程

```
用户点击保存
  ↓
前端校验字段（非空、不重复）
  ↓
更新本地 screenplay_data 对应字段
  ↓
调用 PUT /api/projects/{project_id}/artifacts/screenplay_json
  ↓
后端写入 data/projects/{project_id}/artifacts/screenplay_json_v{NNN}.json
  ↓
返回保存后的 artifact（含 version）
  ↓
前端更新 st.session_state["screenplay_data"]
  ↓
前端同步 storage.update_project() 保存到本地 projects.json
  ↓
显示"保存成功"
```

保存失败时：显示后端错误信息，不静默失败。

## 8. 预览展示与防溢出

- 人物卡片：description 截断 60 字，aliases 只显示前 2 个
- 场景卡片：dramatic_purpose 截断 100 字，action/dialogue 在展开区域显示
- 情节卡片：description 截断 100 字
- 使用 Streamlit 原生组件（st.container、st.write），由框架自动处理换行和宽度
- 不使用固定高度 HTML 卡片，避免溢出
